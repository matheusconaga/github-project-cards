# 🤝 Contribuindo para o GitHub Project Cards

Antes de mais nada, muito obrigado por considerar contribuir para o GitHub Project Cards! 💙 São pessoas como você que fazem da comunidade de código aberto (open-source) um lugar incrível para aprender, inspirar e criar.


## 🛠️ Configuração do Desenvolvimento Local

Siga estas instruções passo a passo para obter uma cópia local do projeto e executá-la em sua máquina.

### 📥 1. Clonar o repositório:
Primeiro, faça um fork do projeto para a sua própria conta do GitHub e, em seguida, clone-o localmente:
```bash
git clone [https://github.com/matheusconaga/github-project-cards.git](https://github.com/matheusconaga/github-project-cards.git)
```

### 📂 2. Entrar na pasta do projeto
Navegue até o diretório recém-clonado:

```bash
cd github-project-cards
```

📦 3. Instalar as dependências
Instale todos os pacotes necessários usando o npm:
```bash
npm install
```
ou
```bash
npm i
```

### ▶️ 4. Executar o servidor de desenvolvimento do projeto
Inicie o servidor local para ver suas alterações em tempo real:

```bash
npm run dev
```

Abra http://localhost:3000 (ou a porta especificada no seu terminal) no seu navegador para visualizar a aplicação.

## 🎯 Formas de Contribuir

Você pode contribuir:

- 🐛 Relatando Bugs: Abra uma issue descrevendo o problema e como reproduzi-lo.
- 💡 Sugerindo Funcionalidades: Compartilhe suas ideias sobre como melhorar este projeto.
- 📝 Melhorando a Documentação: Corrija erros de digitação, esclareça o uso ou traduza seções.
- 🎨 Criando Novos Temas: Adicione novas folhas de estilo/predefinições para os cartões SVG.
- ⚡ Performance & Refatoração: Limpe a base de código ou aumente a velocidade de carregamento.


## 🚀 Guia Passo a Passo para Pull Requests

Quando estiver tudo pronto para enviar suas alterações, siga estas práticas recomendadas:

### 1. Fazer o Fork do Repositório: Garanta que você tem o seu próprio fork do repositório principal.

### 2. Criar uma Branch para a Funcionalidade: Escolha um nome de perfil para a sua branch.

```bash
git checkout -b profileexample
```

### 3. Commitar suas Alterações: Mantenha suas mensagens de commit claras, concisas e em inglês.

```bash
git commit -m "feat: add Neon theme option"
```

### 4. Dar Push para o seu Fork: Envie a branch para o seu repositório pessoal do GitHub.

```bash
git push origin profileexample
```

### 5. Abrir um Pull Request:
Acesse o repositório original no GitHub, vá até a aba "Pull Requests" e clique em "New pull request". Explique claramente suas alterações no modelo de descrição.


## 👥 Adicionando você mesmo à lista de Usuários

Se você está usando o GitHub Project Cards no seu perfil ou portfólio, sinta-se à vontade para se adicionar à seção de Usuários no README.md!

Basta adicionar o link do avatar do seu perfil do GitHub e a URL do seu perfil com hiperlink na área designada dentro das alterações do seu PR:

```bash
<a href="[https://github.com/your-username](https://github.com/your-username)">
  <img src="[https://github.com/your-username.png](https://github.com/your-username.png)" width="90px" alt="your-username"/>
</a>
```
