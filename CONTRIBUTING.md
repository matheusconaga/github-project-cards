# 🤝 Contributing to GitHub Project Cards

First off, thank you for considering contributing to GitHub Project Cards! 💙 It's people like you who make the open-source community an amazing place to learn, inspire, and create.

## 🛠️ Local Development Setup

Follow these step-by-step instructions to get a local copy of the project up and running on your machine.

### 📥 1. Clone the repository:
Fork the project to your own GitHub account first, then clone it locally:
```bash
git clone [https://github.com/matheusconaga/github-project-cards.git](https://github.com/matheusconaga/github-project-cards.git)
```

### 📂 2. Enter the project folder
Navigate into the newly cloned directory:
```bash
cd github-project-cards
```

### 📦 3. Install dependencies
Install all the required packages using npm:
```bash
npm install
```
or 
```bash
npm i
```

### ▶️ 4. Run the project development server
Start the local server to see your changes in real-time:
```bash
npm run dev
```

Open http://localhost:3000 (or the port specified in your terminal) in your browser to view the application.

## 🎯 Ways to Contribute

You can contribute by:

- 🐛 Reporting Bugs: Open an issue describing the bug and how to reproduce it.
- 💡 Suggesting Features: Share your ideas on how to make this project better.
- 📝 Improving Documentation: Fix typos, clarify usage, or translate sections.
- 🎨 Creating New Themes: Add new stylesheets/presets for the SVG cards.
- ⚡ Performance & Refactoring: Clean up the codebase or boost load speeds.

## 🚀 Step-by-Step Pull Request Guide

When you're ready to submit your changes, follow these standard practices:
### 1. Fork the Repository: Ensure you have your own fork of the main repo.

### 2. Create a Feature Branch: Choose a profile name for your branch.
```bash
git checkout -b profileexample
```

### 3. Commit Your Changes: Keep your commit messages clear, concise, and in English.
```bash
git commit -m "feat: add Neon theme option"
```

### 4. Push to Your Fork: Send the branch up to your personal GitHub repository.
```bash
git push origin profileexample
```

### 5. Open a Pull Request:
Head over to the original repository on GitHub, go to the "Pull Requests" tab, and hit "New pull request". Clearly explain your changes in the description template.

If you are using GitHub Project Cards on your profile or portfolio, you are welcome to add yourself to the Users section in the README.md!

Simply add your GitHub profile avatar link and hyperlinked profile URL to the designated area inside your PR changes:
